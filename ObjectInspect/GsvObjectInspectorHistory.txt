История модификаций GsvObjectInspector

Версия 1.01
1) Добавлена генерация исключительных ситуаций при
   неправильном вводе ColorRGB
2) Добавлена история инспектируемых типов для обоих
   значений HideReadOnly
3) Ресурсы графических образов инспектора перенесены из
   dcr-файла в отдельный res-файл
4) Исправлена неточность определения длинного хинта для
   свойств типа pkFolder в том случае, если FolderFontStyle
   отличен от [ ]
5) В визуальный компонент добавлен public метод SmartInvalidate.
   В отличие от Invalidate, метод SmartInvalidate обновляет инспектор
   только в том случае, если у инспектируемого объекта изменилось
   одно или несколько свойств, отображаемых в данный момент
   инспектором. Поле редактирования обновляется только в
   том случае, если редактируемый текст не был изменен, то есть,
   не находится в процессе редактирования. Метод SmartInvalidate
   можно вызывать из события Application.OnIdle, по какому-либо
   таймеру или по какому-либо другому событию

Версия 1.02
1) Исправлена ошибка - передача фокуса ввода инспектору при вызове
   метода NewObject
2) Добавлено свойство визуального компонента инспектора - InplaceEditor,
   свойство доступно только по чтению и возвращает объект редактора свойств,
   приведенный к TCustomEdit

Версия 1.03
1) В предыдущем исправлении ошибки я слишком сильно размахнулся и
   заодно с убиранием фокуса при NewObject убрал слишком много, так что
   инспектор перестал правильно позиционировать поле ввода при работе с
   клавиатурой. Вот этот широкий размах и исправлен в версии 1.02
2) Код приведен к виду, совместимому с Delphi 5, то есть, код должен
   компилироваться без ошибок и предупреждений в Delphi 5, 6, 7
   (спасибо Илье Кирову ikirov@bars-it.ru)

Версия 1.1
1) Исправлена ошибка - отсутствие реакции на PgUp, PgDown при небольшом
   числе полей инспектора
2) Добавлена реакция на клавиши Home, Ctrl+Home, End, Ctrl+End в поле
   редактирования
3) Исправлена ошибка потери фокуса при щелчке мышкой на хинте
4) Исправлена ошибка, возникающая при анализе пустого набора метаданных
   (Илья Киров)
5) Добавлено переключение значения поля pkBoolean по клавишам Space, Enter
   (Илья Киров)
6) Добавлено раскрытие или скрытие полей типа pkSet по клавишам Space, Enter,
   Left, Right
7) Добавлено раскрытие или скрытие полей типа pkFolder по клавишам Left, Right
8) В дескриптор метаданных добавлено поле EditMask, позволяющее задать маску
   ввода строкового значения. Маска используется только для организации ввода -
   введенное значение соответствует результату вместе с литеральными символами.
   Второе поле маски должно быть равно 1 (Илья Киров)
9) Добавлен новый тип свойства - pkFloat, который позволяет форматировать
   значение вещественного числа отдельно для положительных, отрицательных и
   нулевого значения. Для этого типа в дескрипторе определено новое поле
   FloatFormat (Илья Киров)
10) Добавлен новый тип свойства - pkTextDialog, который позволяет ввести
   строку как напрямую, так и с помощью диалога (Илья Киров)

Версия 1.11
1) Исправлена ошибка, внесенная при в версии 1.1 для реализации раскрытия
   поля типа pkSet по клавише пробела. Симптом - в поле ввода нельзя было ввести
   символ пробела
2) Исправлена ошибка, которая появлялась при свертывании дерева свойств.
   Условия ошибки - а) дерево свойств сворачивалось по (+), б) после
   свертывания все свойства могли помещаться в окне без скроллинга,
   в) верхним видимым свойством было не первое свойство. В результате
   ошибки полоска скроллинга отсутствовала, но отображались не
   все свойства (то есть, позиция верхнего свойства не корректировалась).

Версия 1.12
1) Изменена сигнатура метода класса TGsvObjectInspectorTypeInfo.FillList.
   Первый аргумент теперь имеет тип PGsvObjectInspectorPropertyInfo вместо
   TObject. Такая сигнатура более удобна при динамическом порождении
   инспектируемых свойств объекта
2) Исправлена ошибка отображения очень длинного хинта

Версия 1.13
1) Добавлено сворачивание-разворачивание вложенных свойств по клавишам
   + и - (спасибо Сергею Галездинову sega-zero@yandex.ru).
2) Исправлена ошибка, выражающаяся в исключительной ситуации при
   ошибочном вводе текста с использованием маски (Сергей Галездилов).
3) В визуальном компоненте TGsvObjectInspector добавлена процедура
   ValidateStringValue, позволяющая перехватить и обработать исключительную
   ситуацию ошибочного ввода текста с использованием маски. Эту процедуру
   можно вызывать в обработчике OnSetStringValue перед вызовом процедуры
   SetStringValue.

Версия 1.14
1) Добавлено событие OnGetCaption. Основное назначение этого события -
   использование ресурсных строк вместо константных строк. Для использования
   этой возможности в INFO-описаниях классов нужно: а) в Caption задавать имена
   ресурсных строк (а не названия свойств), б) определить обработчик, который
   должен возвращать строку по имени ресурса.
2) Добавлена поддержка XP-стилей

Версия 1.15
1) Добавлена обработка события WM_HELP

Версия 1.16
1) Устранена ошибка, связанная с выпадающим окном списка. При переключении
   на другую задачу окно списка оставалось видимым. Для устранения этой ошибки
   добавлена обработка сообщения WM_KILLFOCUS, так как OnExit в этом случае
   не вызывается (спасибо Максиму Гвоздеву max@papillon.ru).

Версия 1.17
1) Добавлено свойство Layout, позволяющее сохранить в виде строки
   (и затем восстановить) текущий вид инспектируемого объекта
   (выделенное свойство, верхнее видимое свойство и состояние
   раскрытых-закрытых сложных подсвойств). Чтение Layout возвращает строку
   специального формата, а установка Layout восстанавливает состояние
   инспектора из сохраненной строки. При восстановлении инспектор должен
   инспектировать тот же самый объект (по крайней мере, объект того же типа),
   и, кроме того, свойство HideReadOnly не должно быть изменено.

Версия 1.18
1) Добавлена функция Selected, которая возвращает описатель выделенного свойства.
2) Добавлена функция SelectedText, которая возвращает значение выделенного свойства.
2) Добавлена процедура SetSelected, которая устанавливает выделенное свойство
   по его имени.

Версия 1.19
1) Исправлена ошибка неправильного отображения полоски скроллинга при изменении
   размеров компонента
